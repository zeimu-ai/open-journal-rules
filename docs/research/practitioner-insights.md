## 3. 自動仕訳で「間違いやすい」「判断が分かれる」ケース

### 間違いやすいケース

| ケース | 詳細 | Xでの声 |
|--------|------|---------|
| **クレカ+通販の二重計上** | Amazon注文とクレカ引落の両方を仕訳 → 二重計上 | MF公式が回避方法をnoteで解説するほど頻出 |
| **消耗品費 vs 備品** | 10万円未満=消耗品費、10万円以上=備品（資産計上） | 「金額による勘定科目変更」の代表例 |
| **交際費 vs 会議費** | 1人5,000円以下=会議費、超=交際費（法人税法基準） | @TOMOAKIishimaru: 金額分岐対応が必須 |
| **事業用 vs 個人用** | 同じATM出金でも用途で科目が変わる | @fukushimajicpa: 「表参道」が事業主借判定される問題 |
| **振込手数料の差額** | 請求額10,000円に対し入金9,560円 → 差額440円の処理 | MF公式が自動仕訳ルールTipsとして紹介 |

### 判断が分かれるケース

| ケース | 選択肢 | 判断基準 |
|--------|--------|---------|
| **開業費の計上** | 一括経費 vs 繰延資産 | 10万円以上/未満、5年均等or任意償却 |
| **借入金返済の仕訳** | 元本+利息一括 vs 分離 | 顧問先の希望・税務上の要件 |
| **売上の計上時期** | 発生主義 vs 現金主義 | 法人=発生主義（原則）、個人=選択可 |
| **消費税の区分** | 課税/非課税/不課税/免税/軽減税率 | 取引内容・相手先・金額による判定が複雑 |

## 4. freee/MFの自動仕訳に対する不満・要望

### freeeへの不満

| 不満 | 投稿者 | 詳細 |
|------|--------|------|
| **過去データの悪影響** | @fukushimajicpa | 過去の誤った仕訳パターンを学習し、新しい取引でも誤った科目を推定 |
| **素人向け自動化のリスク** | 匿名 | 自動化が素人には早すぎ、確認なしに「ゴミ帳簿」を生む |
| **直接仕訳の方が速い場合** | 匿名 | OCR+推定のステップを踏むより手入力の方が速いケースがある |

### MFへの不満

| 不満 | 投稿者 | 詳細 |
|------|--------|------|
| **同一パターンでの科目誤判定** | @u1tnk | 同じ入出金パターンなのに科目判定がブレる → ストレスで退会 |
| **ルール設定の手間** | 複数 | 自動仕訳ルールの初期設定が面倒。テンプレートがない |

### 共通の要望

| 要望 | 詳細 |
|------|------|
| **業種別テンプレート** | 業種ごとの標準仕訳ルールをプリセットで提供してほしい（石丸氏の記帳AIエージェントが対応） |
| **イレギュラー除外** | 定型取引以外を自動仕訳対象外にしたい |
| **金額分岐ルール** | 10万円以上は備品、5,000円超は交際費等の自動判定 |
| **消費税自動判定** | 取引内容から課税区分を自動判定してほしい |
| **証憑との紐付け** | 仕訳と領収書の自動マッチングが不完全 |

## 5. 税理士が「これは自動化できる」と考えているパターン

Xでの声をもとに整理:

| パターン | 自動化の確信度 | 条件 |
|---------|:----------:|------|
| 毎月定額の入出金（家賃・保険・通信費） | ◎ | 金額と取引先が固定 |
| 銀行口座の入出金仕訳 | ◎ | 取引先名+金額パターンで判定可能 |
| クレカ明細の仕訳 | ○ | 重複チェックロジックが必要 |
| レシート/領収書からの仕訳 | ○ | OCR精度+勘定科目推定の組み合わせ |
| 消費税区分の判定 | △ | 取引内容の理解が必要だがAIで可能（Claudeで実証済み） |
| 元本・利息の分離 | × | 返済予定表との突合が必要 |
| 交際費/会議費の判定 | × | 用途・参加者の情報が必要（伝票の記載に依存） |

## 6. 金額による勘定科目変更の実務ルール

| 金額基準 | ルール | 備考 |
|---------|--------|------|
| **10万円未満** | 消耗品費（即時経費） | 少額減価償却資産 |
| **10万円以上〜20万円未満** | 一括償却資産（3年均等償却）or 備品 | 青色申告なら30万円未満まで即時経費可 |
| **20万円以上〜30万円未満** | 青色申告の場合、少額減価償却資産（即時経費） | 年間300万円上限 |
| **30万円以上** | 固定資産（減価償却） | 耐用年数に応じて |
| **1人5,000円以下（飲食費）** | 会議費 | 法人のみ（2024年4月以降は1万円以下に引上げ予定だったが未定） |
| **1人5,000円超（飲食費）** | 交際費 | 800万円上限（中小法人） |
| **100万円以上** | 安全ルール: 強制要判断 | 高額取引のリスク管理として |

## 7. デフォルトプリセットへの示唆

### 推奨プリセットルール（即時導入可能）

| # | ルール名 | 条件 | 借方 | 貸方 | 根拠 |
|---|---------|------|------|------|------|
| 1 | 通信費（定額） | 取引先名に「NTT/docomo/au/SoftBank/KDDI」+ 毎月同額 | 通信費 | 普通預金 | 定型取引 |
| 2 | 水道光熱費 | 取引先名に「電力/ガス/水道」 | 水道光熱費 | 普通預金 | 定型取引 |
| 3 | 地代家賃（定額） | 取引先名が固定 + 毎月同額 + 「家賃/賃料」含む | 地代家賃 | 普通預金 | 定型取引 |
| 4 | 消耗品費（少額） | 金額 < 100,000円 + EC系（Amazon/ヨドバシ/楽天） | 消耗品費 | 普通預金 | 金額基準 |
| 5 | 備品（中額） | 100,000 ≤ 金額 < 300,000 + 物品購入 | 工具器具備品 | 普通預金 | 金額基準（青色30万特例） |
| 6 | 保険料（定額） | 取引先名に「保険/共済」+ 毎月同額 | 保険料 | 普通預金 | 定型取引 |
| 7 | 振込手数料 | 金額 ≤ 880円 + 「手数料/振込料」含む | 支払手数料 | 普通預金 | 金額基準 |
| 8 | 給与 | 毎月同日 + 取引先名が従業員名 + 一定範囲の金額 | 給料手当 | 普通預金 | 定型取引 |

### 要判断にすべきケース（プリセットで自動判断しない）

| ケース | 理由 |
|--------|------|
| 金額100万円以上の取引 | 高額取引リスク（既に実装済み） |
| 交際費 vs 会議費の判定 | 用途・参加者情報が必要 |
| 借入金返済の元本・利息分離 | 返済予定表が必要 |
| 売上の計上時期判定 | 顧問先ごとのルールに依存 |
| 個人用 vs 事業用の判定 | 証拠書類・用途確認が必要 |

### 重要な設計指針

1. **「ゴミ帳簿」防止**: 確信度が低い仕訳は必ず税理士の確認を求める（freeeの失敗から学ぶ）
2. **過去データの悪影響防止**: 誤った仕訳パターンを学習しないよう、税理士が承認したフィードバックのみを学習対象にする（既にteacher_feedbacks設計で対応済み）
3. **業種別テンプレート**: 顧問先の業種に応じてプリセットルールを出し分ける（飲食業は仕入/食材、IT業は外注費等）
4. **二重計上チェック**: クレカ+通販の重複仕訳を検知するロジックが必須
5. **金額分岐の自動適用**: 10万円/30万円/100万円の境界で勘定科目を自動切替
