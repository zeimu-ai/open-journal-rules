# freee勘定科目体系調査 — API構造・マッピング表

## 背景
ぜいみーさんの自動仕訳プリセットで提案する勘定科目が、freeeの勘定科目と一致する必要がある。マッピングを作るため、freeeの勘定科目体系を調査した。

---

## 1. freee API — 勘定科目エンドポイント構造

### GET /api/1/account_items レスポンスフィールド

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | integer | 必須 | 勘定科目ID（min:1） |
| `name` | string | 必須 | 勘定科目名（30文字以内） |
| `tax_code` | integer | 必須 | 税区分コード |
| `default_tax_code` | integer | 必須 | デフォルト設定の税区分コード（base_dateで変動） |
| `account_category` | string | 必須 | 勘定科目カテゴリー（中分類。例: "現金・預金"） |
| `account_category_id` | integer | 必須 | 勘定科目カテゴリーID |
| `categories` | string[] | 必須 | カテゴリー階層（大分類。例: ["資産"]） |
| `available` | boolean | 必須 | 使用設定（true/false） |
| `walletable_id` | integer | 必須 | 口座ID（nullable） |
| `shortcut` | string | 任意 | ショートカット1（20文字以内、nullable） |
| `shortcut_num` | string | 任意 | ショートカット2（20文字以内、nullable） |
| `code` | string | 任意 | 勘定科目コード（nullable）※事業所設定で有効時のみ |
| `group_name` | string | 任意 | 決算書表示名・小カテゴリー（nullable） |
| `group_id` | integer | 任意 | 決算書表示名ID（nullable） |
| `corresponding_income_name` | string | 任意 | 収入取引相手勘定科目名（nullable） |
| `corresponding_income_id` | integer | 任意 | 収入取引相手勘定科目ID（nullable） |
| `corresponding_expense_name` | string | 任意 | 支出取引相手勘定科目名（nullable） |
| `corresponding_expense_id` | integer | 任意 | 支出取引相手勘定科目ID（nullable） |
| `accumulated_dep_account_item_name` | string | 任意 | 減価償却累計額勘定科目名（法人のみ） |
| `accumulated_dep_account_item_id` | integer | 任意 | 減価償却累計額勘定科目ID（nullable、法人のみ） |
| `items` | array | 任意 | 紐づく品目（id, name） |
| `partners` | array | 任意 | 紐づく取引先（id, name） |
| `searchable` | integer | 必須 | 検索可能:2 / 検索不可:3 |

### クエリパラメータ
- `company_id`（必須）: 事業所ID
- `base_date`（任意）: 基準日。指定した場合、`default_tax_code` がその日付の税率に基づいて返る
- `start_update_date` / `end_update_date`: 更新日絞り込み

### account_item_id は事業所ごとに異なるか

**事業所ごとに異なる。** IDは各事業所内で一意であり、`company_id` を指定してGETする仕組み。異なる事業所間でIDを流用することはできない。

→ **ぜいみーさんへの影響**: プリセットでは `id` ではなく `name`（勘定科目名）でマッチングする必要がある。

## 3. freeeの消費税区分コード（tax_code）体系

税区分コードは3つの時代に分かれて追加されている。

### 3-1. 旧税率時代（〜2019年9月）: コード帯 1〜128

| 用途 | 説明 |
|------|------|
| 課税売上 | 標準税率の課税売上 |
| 課税仕入 | 標準税率の課税仕入 |
| 非課税売上 | 金融・保険・医療等 |
| 非課税仕入 | 土地取得等 |
| 不課税（対象外） | 給与・損害賠償等 |
| 輸出免税 | 輸出取引 |

※旧コード帯の具体的な番号は `GET /api/1/taxes/companies/{company_id}` で取得するのが正規の方法。

### 3-2. 2019年10月追加（消費税2率対応）: コード 129〜182

| コード | name | name_ja | 税率 |
|--------|------|---------|------|
| 136 | purchase_with_tax_10 | 課対仕入10% | 10% |
| 137 | purchase_with_no_tax_10 | 非対仕入10% | 10% |
| 138 | purchase_with_common_10 | 共対仕入10% | 10% |
| 155 | taxable_10 | 課税10% | 10% |
| 163 | purchase_with_tax_reduced_8 | 課対仕入8%（軽） | 8%軽減 |
| 164 | purchase_with_no_tax_reduced_8 | 非対仕入8%（軽） | 8%軽減 |
| 165 | purchase_with_common_reduced_8 | 共対仕入8%（軽） | 8%軽減 |
| 182 | taxable_reduced_8 | 課税8%（軽） | 8%軽減 |

### 3-3. 2023年10月追加（インボイス制度対応）: コード 183〜230

免税事業者等からの課税仕入に対応する経過措置コード:
- **控80%**: コード 183〜206（2023/10/1〜2026/9/30）
- **控50%**: コード 207〜230（2026/10/1〜2029/9/30）

### 3-4. ぜいみーさんで使用頻度が高いコード

| 用途 | コード帯 | 備考 |
|------|---------|------|
| **課対仕入10%** | **136** | 最も頻出。経費の大半 |
| **課対仕入8%（軽減）** | **163** | 食品・飲料関連 |
| 非課税仕入 | 旧コード帯 | 土地・保険・利息等 |
| 不課税（対象外） | 旧コード帯 | 給与・社保・租税公課等 |
| 課対仕入10%（控80） | 183〜206内 | 免税事業者からの仕入（経過措置） |

**重要**: 旧コード帯（非課税・不課税）の正確な番号は事業所ごとに `GET /api/1/taxes/companies/{company_id}` で取得する必要がある。ハードコーディング不可。

## 4. freeeのデフォルト勘定科目

### 件数
公式非公開。実測ベースで法人一般用で**100〜150件程度**が報告されている。業種テンプレート（一般・製造業・不動産等）により異なる。

### freee独自の科目
標準勘定科目体系（企業会計原則）に準拠しており、**freee独自の科目は原則存在しない**。ただし以下のfreee固有概念がある:
- **「対象外」税区分**: 消費税の「不課税」をfreeeでは「対象外」と表記する場合がある
- **口座連動科目**: 銀行口座・クレカと紐づいた科目は `walletable_id` が設定される
- **「勘定科目コード」**: 任意の英数字コード（code フィールド）。デフォルトはNULL

## 6. ぜいみーさんプリセットへの実装上の示唆

### 6-1. 勘定科目のマッチング方式

- **`name`（勘定科目名）でマッチング**: `id` は事業所ごとに異なるため使用不可
- **初回起動時にAPIで科目一覧を取得**: `GET /api/1/account_items` で実際の科目名・IDを取得し、プリセットの科目名と突合
- **名称の揺れに注意**: 国税庁「給料賃金」↔ freee「給料手当」、「外注工賃」↔「外注費」、「利子割引料」↔「支払利息」等

### 6-2. 消費税区分の設定方式

- **新コード（136/163）をデフォルトに**: 課対仕入10%=136、課対仕入8%（軽減）=163
- **旧コード（非課税・不課税）は動的取得**: `GET /api/1/taxes/companies/{company_id}` で取得し、`name_ja` でマッチング
- **科目→税区分のデフォルト紐づけ**: freee APIの `default_tax_code` をそのまま利用するのが最も安全

### 6-3. 名称マッピングテーブル（要実装）

```
国税庁名称 → freee名称
給料賃金 → 給料手当
外注工賃 → 外注費
利子割引料 → 支払利息
貸倒金 → 貸倒損失
税理士等報酬 → 支払報酬
```

上記5つ以外は国税庁名称 = freee名称で一致する。
